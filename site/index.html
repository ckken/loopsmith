把 AI 修复放进可审计的本地循环。

Loopsmith 不直接让模型覆盖源文件。它复制目标 artifact 到候选工作区， 记录每轮输入输出，再用验证命令决定是否进入下一步。

01

Plan

读取配置中的 artifact、goal、profile、模型与验证命令。

02

Review

输出结构化 findings，保留 prompt、schema、answer 和日志。

03

Repair

只在 runs 目录下的候选 workspace 进行 bounded repair。

04

Verify

运行 cargo test、npm test、pytest 或你的项目验证命令。

05

Apply

inspect、diff、dry-run 之后，再显式写回并重新验证。

Lifecycle hooks 让工作流接入真实工程门禁。

每个 hook 都会写入 command、stdout、stderr 和 result.json，适合接入格式检查、 单测、代码扫描、发布前检查和失败通知。

pre_run

运行前先确认工作区状态，例如 `git diff --check` 或依赖检查。

post_iteration

每轮结束后追加验证或审计，失败会进入 remaining_delta。

pre_apply / post_apply

写回前后都能跑门禁，保证人工 apply 仍然有工程级防线。

多 reviewer，单 writer。

`multi-review` profile 会触发 correctness、tests、docs 三个只读 reviewer， findings 合并后只交给一个 repair writer，降低并行改写冲突。

安装后直接在本地项目里跑。

Loopsmith 是 Rust 单二进制 CLI。它包装 Codex CLI，但最终通过你的验证命令、 diff 和 apply 决定是否接受修复。

最小 vibecoding 工作流

先生成候选修复，再 inspect、diff、dry-run。只有人工确认后才 apply 回源工作区。
